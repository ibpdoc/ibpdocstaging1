Create a Check In object

When you used the Quick Create wizard to create the app, you were able to create multiple objects and add fields and relationships to them in one flow. When you create an object explicitly, you first create the object definition and then add components such as fields and relationships. Because each Check In record will basically be a copy of a Check Out record, the Check In object should have the same fields and relationships. Follow these steps to add a Check In object and add the appropriate fields and relationships:

  1. Create the object definition:

    1. From the Platform menu, select New Menu.

      The What do you want to create? dialog opens.

    2. Leave the default selection, A new Object (with Tab), and click Create.
      The New Object screen opens.

    3. For the Singular Name, enter Check In.
    4. Click Save.
      The New Tab screen opens.

    5. In the User Roles table, make sure Administrator and Library Employee are selected. Library Member should not be selected because members should not be able to view these records.
    6. Leave the default values for the other fields.
    7. Click Save.
    A new Check Ins page opens in the application.
  2. Add two date fields to Check In in the object definition:
    1. From the page menu, select Object definition.
    2. From the shortcut bar, select Fields.
    3. Click New Field.
      The New Field screen opens.
    4. For Field Type, select Date and click Next.
    5. Enter the following values:
      • For Field Label, Date Borrowed
      • For Integration Name, ci_date_borrowed (adding "ci" makes it easier to distinguish between this field and the one with the same name in the Check Out object)
      • Leave the other fields with the default settings.
    6. Click Save.
      The Add to Pages and Views screen opens.
    7. Leave the default boxes checked and select the box to add this field to the All Check Ins view.
    8. Click Save & New.
    9. Use the same procedure to create a Date field with the name Date Due and an integration name of ci_date_due. Add it to the All Check Ins view. Click Save & New.
    10. Use the same procedure to create a Date field with the name Date Returned and an integration name of ci_date_returned. Add it to the All Check Ins view and save it.
  3. Create a relationship between the Check In and Title objects:
    1. Use the shortcut bar to jump to the Relationships section of the Check In object definition.
    2. Click New Relationship.
    3. Choose Title as the related object, and click Next.
    4. For Cardinality, leave the default, Many Check Ins to Many Titles. Since a particular title will be checked out and back in repeatedly, this cardinality allows the same title to be attached to multiple Check In records.
    5. In the Add Lookup Fields to Pages section, click Select All for both Check In Pages (Title lookup field) and Check In Pages (Titles related list view).
    6. In the Add Related List Views to Pages section, click Select All under Check In Pages (Title lookup field) and Check In Pages (Titles related list view).
    7. Save the relationship.
  4. Use the same procedure to add a relationship between Check In and User objects. Set the cardinality to Many Check Ins to One User, and leave the default values for Add Lookup Fields to Pages (all Check In pages are selected and User Pages are not).
  5. Return to the application Check Ins page and edit the All Check Ins view to include the fields associated with these relationships:
    1. From the Viewing: All Check Ins menu, select Edit this View.
    2. Move the Titles and Users fields from the Available Columns list to the Selected Columns list.
    3. Use the up and down arrows to arrange the columns in a logical order.
    4. Save your changes.
