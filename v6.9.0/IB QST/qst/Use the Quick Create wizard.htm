Use the Quick Create wizard

With the objects and fields in his diagram as a reference, Adam used the Quick Create wizard to create his app foundation. For the Title object, Adam didn't know whether he wanted to create separate fields for authors, actors, and artists. So, he decided to focus on books in his prototype and add one Author(s) field. This modified diagram lists the objects and fields he created along with their data types.

To use the Quick Create wizard to create a similar app foundation, log into your Platform account and follow the steps below.

  1. Launch the Create a New Application dialog:
    1. From an application page, such as Platform Home, open the Platform menu. Click the thumbnail to see the menu location:

    2. Click New Application:

    The Create a New Application dialog opens.

  2. In the New Web Application section of the screen, click Guide Me Through It.
    The Quick Create wizard opens:

     

  3. In the Application Name field, enter Lending Library and click Next.
    The Create Objects page opens.
  4. Add the two application objects, using the singular form of their names:
    1. In the Object Name field, enter Title.
    2. Click Add Object
    3. Enter Check Out as the second object name.
      The Attributes column allows you to add predefined fields and behavior to an object. Neither of these objects use these special built-in attributes, so you do not need to select any.
    4. Click Next.
    The Add Fields page opens.
  5. Add fields to the Title object as follows:
    1. In the Field Name box, enter Author(s).
    2. For Field Type, select Text.
    3. Leave Field Values as is.
    4. Click Add Field to add the remaining fields:
      • Name — PublisherType — Text
      • Name — Publication DateType — Date
  6. In the left pane, select the Check Out object to add the following fields:
    • Name — Date BorrowedType — Date
    • Name — Date DueType — Date
    • Name — Date ReturnedType — Date
  7. Click Next.
    The Relationships page opens.
  8. Enter the relationship between Title and Check Out:
    1. In the infographic, identify the relationship that matches the library use case: many titles can belong to many check out records. The notation for this is M:M.
    2. In the Related To column for the Title object, click Select.
      A dialog opens allowing you to add relationships:
    3. Click an arrow next to Check Out to set the value to M:M and click plus (+) to add the relationship.

    4. Move your cursor out of the dialog and note that Platform automatically added a corresponding relationship from Check Out to Title for you.

  9. Click Submit.
    Platform creates the application foundation and displays a summary. Platform created pages for each object.
  10. Click View your Application.
    The Titles page contains built-in controls that allow end users to add, modify, and view records. If your browser window is very narrow and opens without the Platform menu on the left, click the menu icon to open it.
  11. Click the C icon to view the Check Outs page.

  12. Click the menu icon to expand and collapse the menu.

Next, Adam created User records and roles to manage the login accounts for library employees and members.
Before testing your app foundation, create roles, accounts, and add the User object to the application as described in Setting up user access and testing.
