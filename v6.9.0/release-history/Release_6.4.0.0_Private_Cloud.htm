Known Issues

Following is the list of known issues with this release:

  1. [Triggers] - Post 6.4 upgrade, navigating to System Console > Monitoring section could throw "Unknown job type" exception.

    Workaround: In such scenario, please contact IB Support for further assistance to clean-up the stale entries in the database.

  2. [Quick Filters] - The Object Type field is not listed in the list view’s quick filters.

    Workaround: Choose the filter on the Object Type in dynamic filters.

  3. [Quick Filters]- List view’s quick filtering throws the following error when the dynamic filter has expression join type -com.rb.core.data.DataException: Cannot add filters to expression-based View

  4. [Quick Filters] - List view’s quick filtering with Dynamic Filter formula join type does not return expected result set.

  5. [Quick Filters] - List view’s quick filtering throws the following exception when quick filter template fields are deleted from the object definition.

    java.lang.IllegalStateException: Illegal call to key() for current state: ARRAY

    Workaround: Open the quick filter template and click on Save to update the template with correct fields.

  6. [Quick Filters] - List view’s keyword filtering with time format throws the following exception.

    Error instantiating a232 for the given a215 instance

    java.lang.Exception: Unable to compute ListView a811

  7. [Quick Filters] - List view’s keyword filtering input text does not have cancel icon to reset the data.

  8. [Quick Filters] - From the List View > Design this page > Select Quick Filter Template option is not enabled/disabled based on other property ‘Enable Quick Filtering’.

  9. [Quick Filters] - List View’s quick filtering with date fields throws client-side validation error when TODAY is used in the filter input.

    Workaround: Use Date Interval property in quick filter template to use TODAY keyword token.

  10. [Quick Filters] - List view page reload clears quick filter criteria from UI.

    Workaround: Use "Remove Filters" button to reset quick filters.

  11. [Quick Filters] - Quick Filtering does not work for look-up dialog box.

  12. [Backup & Restore] - On uploading a backup, error "User Session is not found" is thrown intermittently.

    Workaround: Delete "storageSessionIdMaster" cookie manually and then re-upload backup file.

  13. [Record Activities] - If a look-up dialog box is open on a record edit page and the page is saved without closing the dialog box, it still remains open.

    Workaround: Close the look-up dialog box manually (or) reload the list view page.
