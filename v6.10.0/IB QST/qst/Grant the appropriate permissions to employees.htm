Grant the appropriate permissions to employees

The built-in User object is associated with the Library Members tab of the application and library employees need to view user records to associate them with check out records. There are several ways to grant permissions to library employees, as an individual or by role. Adam decided to grant permission to the Library Employee role for viewing, creating, and editing User records and view permissions for the All Users view.

To enable library employees to manage library member records, follow these steps:

  1. From the application switcher, select Platform Home.

  2. From the application menu bar, expand Users and select Roles.

  3. In the Role table's Library Employee row, click Permissions.

    The Permissions page opens and displays the role's permissions.

  4. From the Filter by Application menu (at the top of the page), select Platform (because the User object is part of the Platform application).
  5. In the Administrative Permissions section, Manage Users row, enable View permission.

  6. Scroll to find the User object, click Select All.

  7. Scroll to Users (in the Tabs section), and click Select All to enable view permissions.
  8. Return to the top of the page and select Lending Library in the Filter by Application field.
  9. Scroll to Library Members (in the Tabs section), and click Select All.
  10. Save your changes.
  11. Log in as a library employee and check your changes by viewing the records on the Library Members page.
When Adam logged in as an employee, he realized that employees can see all User records. They need the permission to create and update User records, but they should only be able to see the records of library members. Similar to the way he created a view for filtering Titles, he created a view to filter these records.
