Add user accounts

If you already have at least three user accounts in your tenant, you can edit them and temporarily assign one user record to the role of Library Employee, and another to the role of Library Member. If you don't have the accounts, follow the steps described in this topic to create them. You can choose any name. This tutorial uses Dewey Decimal for the employee and Evert Reeder for the library member.

Note: Each User must have an email address that is unique within the tenant. You cannot reuse the email address associated with your Platform account. If you do not have two additional email addresses to test with, you can easily create them using a free service such as gmail, yopmail, or yahoo.

After you have the email addresses to associate with the accounts, create the accounts by following these steps:

  1. Navigate to Administration Setup:
    If you are still viewing Roles, in the breadcrumbs, click Administration Setup:

    If you returned to the application Navigate back to Setup Home, from the Application Switcher:

  2. From the Administration Setup section, select Users.
  3. Click New User.

     
    The New User page opens.

  4. To create a user account:
    1. Enter an email address. This must be a real email address but it must be different from the one you used to create your account as an administrator.
      When you save the record, Platform sends an email to this address with the link to log in to the tenant.
    2. Select Library Employee as the User Role.
    3. In the Contact Information section, enter values for the required fields:
      • First Name, such as Dewey
      • Last Name, such as Decimal
      • Phone (can be a fake number)
    4. In the Global Settings section, select a Language and Time Zone.
  5. Click the orange check mark to save the record.
  6. Use the same steps to create a second user account associated with a different email address. Enter a different name and job title, and choose the role of Library Member
