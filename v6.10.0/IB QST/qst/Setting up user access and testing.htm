Setting up user access and testing

Before creating the user account records, Adam learned about Platform tenants — virtual spaces for applications that can be shared with other developers and with end users. Adam's tenant is on Hosted Platform, www.app.infiniteblue.com, which is managed by Infinite Blue. Private Cloud Platform, installed and managed by a Infinite Blue customer or partner, can have multiple tenants. On both Hosted and Private Cloud, a user with the Administrative role allows others to log into the tenant by creating User records for them.

Among other things, a User record includes a login name and an email address. On creation of a new User record, Platform sends a welcome email that contains a link to the Platform login page (the URL is different for Hosted than for Private Cloud tenants), the login name, and a temporary password.

In User records, User Role is a required field. Roles define permissions for a group of users and are central to Platform security and access control. Platform has built-in roles, like Administrator, which provides access to both application and the setup pages that allow you to configure and create applications. Administrators can create as many roles as they need and assign them to users. See Roles & Permissions for more information.

For his Lending Library app, Adam decided on the following roles and the associated permissions:

  • Library Employee — creates, edits, and deletes titles and library members and creates check out records.
  • Library Member — views available titles and their own check outs, but cannot view information about library employees or other library members.
