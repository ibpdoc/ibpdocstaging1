Make it possible to select borrower

The built-in Platform User object stores information about library employees and library members. You already added sample records and a tab to make these visible in the application. To make these records selectable when creating a Check Out record, you need to add a relationship between the Check Out object and the User object. When you add a relationship between objects, Platform adds a lookup field that allows end users to select a related record.

Add the lookup field for library members:

  1. If it is not already open, navigate to the Check Out object definition.
  2. Use the shortcut bar to jump to the Relationships section.
  3. Click New Relationship to open the New Relationship page.
  4. For Related Object, select User and click Next.
  5. Leave the default values in the first four fields.
  6. For Cardinality, since one member can have multiple check outs, select Many Check Outs to one User.
  7. Leave the remaining fields with their default values.
  8. Click Save.
    Platform adds a lookup field for users to the Check Out object definition. Next, add the User relationship to the All Check Outs view.
  9. On the Check Out object definition screen, use the shortcut bar to jump to the Views section.
  10. In the row for the All Check Outs view, click Edit.
  11. In the Columns section, move User to the Selected Columns box, position it after Titles and save the view.
Now your app contains all the functionality necessary to handle check outs. Test the check out process to make sure your app works as designed.
