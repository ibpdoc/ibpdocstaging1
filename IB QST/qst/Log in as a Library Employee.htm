Log in as a Library Employee

To replicate Adam's usability test, log in as a library employee. You can only do this if you created a user account with a role of Library Employee as described in Setting up user access and testing. Once you have such an account, follow these steps:

  1. Do the following to obtain the credentials and log into Platform:

    1. Log into the email account associated with the user and find the Welcome email that Platform sent to that address.

      The email will contain information similar to the following:

    2. Note the user name and temporary password.
    3. Use the Login URL to log in.
  2. Change the password:

    1. Open your profile menu and select My Profile.

    2. Select CHANGE MY PASSWORD, enter the necessary information, and click Save .

      On success, a message displays and the My Profile page opens.

  3. From the Platform menu, select Titles to return to the application.

  4. Notice the following:

    • The banner does not contain an Application Switcher. The role of Library Employee only has permissions to view and use this one app, so there is no way to switch to another app.

    • Similarly, development options, such as creating a new application, are not available from the Platform menu. Only users with the Administrator role can create and edit applications and their components.

    • The user is presented with a page that contains list of titles and no orientation as to what to do first.
    • The Titles tab is the first in sequence from top to bottom, so this tab displays first when users log in. Since creating Check In records will likely be a more common tasks, it makes sense to change the order of the tabs.

  5. Select the Check Outs tab and click New Check Out to create a Check Out record. Note the following:

    • The New Check Out page allows the library employee to select the date fields, but Platform is going to automatically set the date borrowed and date due, and the date returned should not have a value when an employee creates a new record.
    • When you try to select a User (library member), Platform displays an error message because the Library Employee role does not have permission to view or edit user records. Permissions must be changed to allow employees to view and edit these records.
    • Cancel to avoid creating a record with no associated library member.

  6. On the Check Outs page, click the plus control to try the Quick Create functionality.

  7. Notice that the Quick Create page has the following issues:

    • Similar to the New Check Out page, it includes the Date Returned field.
    • It duplicates the functionality a user gets when clicking New Check Out. If the Check Out object contained many more fields and you wanted to provide a way for users to enter values for just a few, the Quick Create button would be very useful. However, in this case it is not necessary and adds a potentially confusing control to the interface.
  8. Since you weren't able to test checking out and in as a Library Employee, log in with your original account (with the Administrator role) and test the process.
    1. Pick one title, check it out to a library member and then check it in. Check the same title out again. Hint: To check in a title, edit the value of the Date Returned field of the Check Out record to add a value.

    2. Select the Check Outs tab and look at the view of all Check Outs

      You will notice that the list of check outs includes both current check outs and check outs that have been checked in. The view could be confusing to library employees, who would expect to see only current check outs. Adam can fix this by modifying the view to only show current check outs.

Adam's list of things to fix include:

  • Creating a page to orient new users
  • Adding a page for member records
  • Modifying application tab order and the fields that display when creating a new check out record.
  • Granting employees the appropriate permissions.
  • Creating a view to show only library member records.
  • Modifying the Check Out view to only include current check out records.
